Design Primers

40 bp of complementary sequence at the termini of substate DNA sequences is sufficient for the Gibson assembly reaction. This can be achieved by adding 20 bp from adjacent substrate sequences.

Note: as little as 20 bp complementary sequence may be sufficient for assembly; this usually comes at a cost to efficiency

Example workflow: Example workflow:
  1. Take 60 bp around each junction between substrate sequences
    • The target primer size will still be approximately 40 bp; starting with 60 bp facilitates optimization because potential secondary structures affecting a range of primers designs can be assessed simultaneously.{% call macros.screenshot( name="ape_primer" ) %}{% endcall %}
  2. Analyse the 60 bp sequences e.g. using IDTâ€™s OligoAnalyzer
  3. Optimize the primer sequences as per standard PCR (see guides for primer design for PCR for further tips). Note that it may be difficult and unnecessary to meet all desirable primer design criteria simultaneously.
    • Parameters to consider optimising:
      1. Overlap length: Try to keep >15nt from each substrate sequence
      2. Melting temperature: Aim to match primer pair Tm values within 4 °C.
      3. Secondary structure: Make sure any hairpins have low Tm (typically less that 40 °C). Any hairpin with Tm over ~30 °C, should be >5bp from 3′ end or primers.
      4. Primer Dimers
    • If any of the above parameters are unacceptable, then consider adjusting the following variables:
      1. Primer length
      2. Primer position relative to junction
  4. Order primers
Gibthon Primer Design

Having designed your construct in the Gibthon Construct Designer, you will be able to automatically generate and design primers.

The primer design engine used by Gibthon scores primers on the following metrics:

  • Melting temperature
  • Mis-priming
  • Self-priming
  • Secondary structures
  • Primer dimers
  • Unique regions of homology

You will then be presented with an overview of the generated primers, along with the option to either fix manually or attempt to automatically fix any detected issues.
